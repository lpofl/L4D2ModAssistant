## 可配置组合策略：
种子：一个随机数种，可roll，可输入
随机组合以下分类：一个带有缩进的二级目录结构的分类表。默认按一级分类折叠，通过多选框选择，选择和取消一级标签时对其下属所有二级标签生效。使用三态复选框。默认勾选生还者、枪械、近战、消耗品。分类表默认折叠，折叠状态下以一行文本显示选中的分类。
评分设置：仅大于等于输入的评分，默认0
优先包含以下标签：一个带有缩进的二级目录结构的标签表，通过多选框选择，选择和取消一级标签时对其下属所有二级标签生效。标签表默认折叠，折叠状态下以一行文本显示选中的标签。
不包含以下标签：一个带有缩进的二级目录结构的标签表，通过多选框选择，选择和取消一级标签时对其下属所有二级标签生效。标签表默认折叠，折叠状态下以一行文本显示选中的标签。
选项：减少同质化MOD一同出现
选项：优先使用高评分。
选项：优先使用低频使用MOD。
选项：新MOD优先。
选项：不使用最近X个已使用方案中使用过的MOD（默认：3） 选项：无可用MOD时允许使用最近使用过的MOD
选项：体积上限：单位GB，默认2.0，与设置页面同步（需要单位换算）

进阶模式按键：暂不实现，只留出接口，效果是显示权重参数供玩家调节


## 随机组合方案
使用Gumbel-Top-k
- 先把原始特征标准化到 [0,1]
    - 评分：ri∈[0,1]（ 0–5 评分除以 5）
    - 使用次数：ui = 1−e^(− use/k) 滑动窗口+非线性压缩，k默认5，默认统计w=90天内的使用数据
    - 优先 TAG：若匹配则 bi=1，否则 bi=0
    - 新MOD优先：最后保存时间在一个月内，且未使用过，则ni=1,否则ni=0
- 计算每个 MOD 的偏好分 S_i：S_i = α·ri + β·ui + γ·bi + δ·ni
    α∈[0,3]：高评分优先强度
    β∈[0,3]：低使用频率优先强度（越大越偏向低频）
    γ∈[0,3]：优先 TAG的额外加成
    δ∈[0,3]：新MOD偏好
    T属于[0,1]
    参数默认：α=1.4, β=0.8, γ=1.0, δ=0.8, T=0.30
- 从Si变成抽样权重wi wi=exp( Si/T )
- Gumbel-Top-k 抽样 key_i = log(w_i) + Gumbel(0,1)
- 每个分类按分数高低取MOD，取的时候排除冲突模组，加入前置MOD和后置MOD。同质MOD只有在该分类下没有其他MOD可用时才会使用。
自定义MOD的从MOD不参与Gumbel-Top-k计算，若主MOD被选中，则每个槽位等概率随机选取或不选取一个从MOD加入生成结果。从MOD不计入方案总体积
若方案超过容量上限，则计算ratio_i=key_i/size_i，从将方案中低ratio_i的MOD替换其分类中的高ratio_i，直到满足容量要求。
若体积最小方案都超过容量要求，则返回容量过小警告
